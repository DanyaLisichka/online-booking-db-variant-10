# Описание предметной области

Разрабатывается база данных для системы онлайн-записи в психологический центр. Система позволяет клиентам записываться на очные и онлайн-консультации, проходить регистрацию, оставлять заявки и взаимодействовать с психологами.

Также предусмотрено:

- хранение списка методик психологов;
- возможность анонимных заявок;
- ведение заметок психолога;
- ограничение на частоту записи клиента к одному психологу.


Выделенные сущности предметной области

## 1. Клиенты (clients)

Хранит информацию о зарегистрированных клиентах центра.

Атрибуты:

- client_id — идентификатор клиента;
- full_name — ФИО клиента;
- phone — телефон;
- email — электронная почта;
- registration_date — дата регистрации.
## 2. Психологи (psychologists)

Содержит информацию о специалистах центра.

Атрибуты:

- psychologist_id — идентификатор психолога;
- full_name — ФИО психолога;
- specialization — специализация.

## 3. Методики (methods)

Справочник методик, используемых психологами.

Атрибуты:

- method_id — идентификатор методики;
- method_name — название методики.
  
## 4. Методики психологов (psychologist_methods)

Промежуточная таблица для связи многие-ко-многим между психологами и методиками.

Атрибуты:

- psychologist_id — идентификатор психолога;
- method_id — идентификатор методики.
  
## 5. Заявки (requests)

Хранит обращения клиентов и анонимных пользователей.

Атрибуты:

- request_id — идентификатор заявки;
- client_id — идентификатор клиента;
- anonymous_name — имя анонимного пользователя;
- request_text — текст обращения;
- request_date — дата заявки.

## 6. Записи на консультацию (appointments)

Основная таблица записей на консультации.

Атрибуты:

- appointment_id — идентификатор записи;
- client_id — идентификатор клиента;
- psychologist_id — идентификатор психолога;
- appointment_date — дата консультации;
- consultation_type — формат консультации (online/offline).

## 7. Заметки психолога (psychologist_notes)

Содержит внутренние заметки специалиста.

Атрибуты:

- note_id — идентификатор заметки;
- appointment_id — идентификатор записи;
- psychologist_id — идентификатор психолога;
- note_text — текст заметки;
- created_at — дата создания.


## Cвязи между сущностями

Один-ко-многим:
Один клиент → много записей;
Один психолог → много консультаций;
Одна запись → много заметок невозможно, поэтому связь один-к-одному;
Один клиент → много заявок.
Многие-ко-многим:
Один психолог может использовать несколько методик;
Одна методика может применяться несколькими психологами.

Для реализации связи многие-ко-многим была создана промежуточная таблица psychologist_methods.

## Проверка нормальных форм
Первая нормальная форма (1НФ)

Все таблицы удовлетворяют 1НФ:

- отсутствуют массивы;
- каждое поле содержит одно значение;
- данные атомарны.

Например:

- телефон хранится одной строкой;
- методики вынесены в отдельную таблицу.

## Вторая нормальная форма (2НФ)

Все неключевые поля полностью зависят от первичного ключа.

Например:

- в psychologist_methods используется составной ключ;
- отсутствуют частичные зависимости.
  
## Третья нормальная форма (3НФ)

Отсутствуют транзитивные зависимости.

Например:

- специализация хранится отдельно у психолога;
- методики вынесены в отдельную таблицу;
- данные консультации не дублируют данные клиента или психолога.

## Решения по нормализации

Для устранения аномалий были выполнены следующие действия:

- методики вынесены в отдельную таблицу;
- создана промежуточная таблица psychologist_methods;
- заметки психологов выделены отдельно;
- заявки отделены от записей на консультации.

## Триггер ограничения записи
Был реализован триггер:
```

```
Триггер запрещает запись одного клиента к одному психологу чаще одного раза в 3 дня.

При нарушении ограничения выводится ошибка:
```
Запись к этому психологу возможна не чаще одного раза в 3 дня
```

# НАПИСАНИЕ SQL-ЗАПРОСОВ
## 1. Запрос с JOIN нескольких таблиц

Вывод информации о консультациях:
```
SELECT
    c.full_name,
    p.full_name,
    a.appointment_date,
    a.consultation_type
FROM appointments a
JOIN clients c
    ON a.client_id = c.client_id
JOIN psychologists p
    ON a.psychologist_id = p.psychologist_id;
```

## 2. Запрос с группировкой и HAVING

Вывод востребованных психологов:
```
SELECT
    p.full_name,
    COUNT(a.appointment_id) AS total_appointments
FROM psychologists p
JOIN appointments a
    ON p.psychologist_id = a.psychologist_id
GROUP BY p.psychologist_id
HAVING total_appointments > 0;
```

## 3. Запрос с подзапросом

Поиск клиентов, посещавших психологов чаще среднего:
```
SELECT DISTINCT c.full_name
FROM clients c
JOIN appointments a
    ON c.client_id = a.client_id
GROUP BY c.client_id
HAVING COUNT(a.appointment_id) >
(
    SELECT AVG(cnt)
    FROM
    (
        SELECT COUNT(*) AS cnt
        FROM appointments
        GROUP BY client_id
    ) AS stats
);
```

Вывод самых востребованных психологов за квартал:
```
SELECT
    p.full_name AS psychologist,
    COUNT(a.appointment_id) AS total_consultations
FROM psychologists p
JOIN appointments a
    ON p.psychologist_id = a.psychologist_id
WHERE QUARTER(a.appointment_date) = QUARTER(CURDATE())
GROUP BY p.psychologist_id
ORDER BY total_consultations DESC;
```
# ПРОВЕРКА ЦЕЛОСТНОСТИ И ОГРАНИЧЕНИЙ
## Проверка триггера

Попытка повторной записи клиента к тому же психологу раньше 3 дней:
```
INSERT INTO appointments
(client_id, psychologist_id, appointment_date, consultation_type)
VALUES
(1, 1, '2026-04-11', 'online');
```
Результат:
```
Ошибка: запись невозможна чаще одного раза в 3 дня
```

## Проверка внешнего ключа
Попытка удалить психолога, у которого есть записи:
```
DELETE FROM psychologists
WHERE psychologist_id = 1;
```
Результат:
```
Ошибка FOREIGN KEY constraint fails
```
## Проверка ограничений

Попытка добавить запись с несуществующим клиентом:
```
INSERT INTO appointments
(client_id, psychologist_id, appointment_date, consultation_type)
VALUES
(999, 1, '2026-05-01', 'offline');
```
Результат:
```
Ошибка нарушения внешнего ключа
```
