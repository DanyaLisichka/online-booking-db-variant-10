## Описание предметной области

Проект представляет собой веб-приложение для психологического центра. Система предназначена для ведения справочников, оформления записей клиентов на консультации, управления статусами записей и формирования отчётов.

В системе используются следующие основные сущности:

- клиенты;
- психологи;
- методики;
- услуги;
- записи на консультацию;
- заявки;
- заметки психологов;
- связи психологов с методиками и услугами.

Клиенты могут записываться на очные и онлайн-консультации. Психологи работают с разными методиками и оказывают разные услуги. Для записей реализованы статусы, проверка занятости временных слотов и генерация кода бронирования.

## Реализованный функционал

### Справочники

В проекте реализованы CRUD-интерфейсы для справочников:

- клиенты;
- методики;
- психологи;
- записи на консультацию;
- услуги.

Для справочников реализованы:

- просмотр списка записей;
- добавление новой записи;
- редактирование существующей записи;
- удаление с подтверждением;
- поиск;
- сортировка по столбцам;
- пагинация;
- flash-сообщения;
- серверная валидация;
- проверка связей перед удалением.

---

### Бронирование консультации

Реализована отдельная бизнес-операция бронирования консультации.

Пользователь выбирает:

1. услугу;
2. психолога;
3. дату;
4. доступный временной слот;
5. клиента;
6. тип консультации.

После выбора услуги список психологов загружается динамически через AJAX. После выбора психолога и даты загружаются свободные временные слоты. Слоты рассчитываются на сервере с учётом рабочего времени и уже существующих записей.

Перед сохранением записи сервер повторно проверяет, свободен ли выбранный слот. Это защищает от двойного бронирования.

---

### Управление записями

Для записей на консультацию реализованы статусы:

- `pending` — ожидает;
- `confirmed` — подтверждена;
- `cancelled` — отменена;
- `completed` — завершена.

На странице списка записей статус отображается цветным индикатором. Также реализованы действия:

- подтвердить запись;
- отменить запись;
- завершить запись;
- редактировать запись;
- удалить запись.

Для списка записей доступны фильтры:

- по тексту;
- по статусу;
- по психологу;
- по диапазону дат.

---

### Отчёты

Реализована страница отчётов:

- количество записей и выручка по дням;
- загруженность психологов;
- выручка по услугам.

Также добавлен экспорт отчёта в CSV-файл.

## Архитектура проекта

Проект реализован с использованием упрощённого MVC-подхода.

### Model

В качестве модели используются репозитории. Они отвечают за работу с базой данных и содержат SQL-запросы.

Примеры репозиториев:

- `ClientRepository`
- `MethodRepository`
- `PsychologistRepository`
- `AppointmentRepository`
- `ServiceRepository`
- `ReportRepository`

### Controller

Контроллеры обрабатывают запросы пользователя, вызывают методы репозиториев и передают данные в представления.

Примеры контроллеров:

- `ClientController`
- `MethodController`
- `PsychologistController`
- `AppointmentController`
- `BookingController`
- `ReportController`

### View

Представления находятся в папке `views`. В них размещена HTML-разметка с минимальными вставками PHP для вывода данных.

SQL-запросы в представлениях не используются.

## Маршрутизация

Маршрутизация реализована через параметры `entity` и `action` в URL.

Примеры маршрутов:

```text
index.php?entity=client&action=list
index.php?entity=client&action=create
index.php?entity=client&action=edit&id=1
index.php?entity=client&action=delete&id=1
index.php?entity=booking&action=create
index.php?entity=report&action=index
```

## База данных

В базе данных используются внешние ключи для обеспечения целостности данных. Также реализован триггер, запрещающий клиенту записываться к одному психологу чаще одного раза в три дня.

Основные таблицы:

- `clients`
- `psychologists`
- `methods`
- `psychologist_methods`
- `services`
- `psychologist_services`
- `requests`
- `appointments`
- `psychologist_notes`

## Инструкция по развёртыванию

1. Клонировать репозиторий:

```bash
git clone https://github.com/DanyaLisichka/online-booking-db-variant-10
```

2. Поместить проект в директорию веб-сервера, например:

```text
/Applications/XAMPP/xamppfiles/htdocs/project
```

3. Создать базу данных в MySQL.

4. Импортировать SQL-скрипт:

```text
schema.sql
```

5. Настроить подключение к базе данных в файле:

```text
config/config.php
```

Пример настройки для XAMPP:

```php
define('DB_HOST', '127.0.0.1');
define('DB_NAME', 'psychology_center');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_CHARSET', 'utf8mb4');
```

6. Запустить Apache и MySQL.

7. Открыть приложение в браузере:

```text
http://localhost/project/index.php
```
